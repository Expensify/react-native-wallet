# UAPP (User Account Payment Provisioning) for Android

## What is UAPP?

UAPP (User Account Payment Provisioning) is an enhanced security flow for Google Wallet that requires encrypted payment credentials during card provisioning. Instead of providing only the TSP Opaque Payment Card (OPC), your backend must also generate a Google Opaque Payment Card (Google OPC) using Google's encryption keys. It allows users to provision cards to Google Wallet and save their cards to their Google Account.

This library handles the UAPP flow automatically. You just need to set up a handler that communicates with your backend to retrieve the encrypted credentials.

> [!IMPORTANT]
> Please familiarize yourself with [official Google docs ](https://developers.google.com/pay/issuers/apis/push-provisioning/android)before starting the implementation!

## How UAPP Works

1. User initiates card provisioning by calling `addCardToGoogleWallet()`
2. During provisioning, Google Wallet requests encrypted credentials
3. The library automatically triggers your registered handler with provisioning details
4. Your handler calls your backend with the server session ID and OPC
5. Your backend generates the Google OPC using Google's encryption keys
6. Your handler returns both the TSP OPC and Google OPC
7. The library automatically provides these credentials to Google Wallet
8. Provisioning continues seamlessly

## Setup

To enable UAPP support, pass a payment credentials handler directly to the `addCardToGoogleWallet()` function:

```typescript
import { addCardToGoogleWallet } from '@expensify/react-native-wallet';

// Define your handler function
const handlePaymentCredentials = async (data) => {
  const { requestId, serverSessionId, walletId, opaquePaymentCard } = data;
  
  // Call your backend to get encrypted credentials
  const response = await fetch('https://your-backend.com/api/wallet/credentials', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      serverSessionId,
      opaquePaymentCard,
      walletId,
    }),
  });
  
  const credentials = await response.json();
  
  // Return the credentials - library will automatically send them to Google Wallet
  return {
    opaquePaymentCard: credentials.tspOpc,
    googleOpaquePaymentCard: credentials.googleOpc,
  };
};

// Pass the handler when adding a card
const status = await addCardToGoogleWallet(cardData, handlePaymentCredentials);
```

## Backend Implementation

> [!NOTE]
> Please refer to the [official Google documentation](https://developers.google.com/pay/issuers/apis/push-provisioning/android/upgrade_to_upp#generate_the_google_opc_from_the_issuers_backend) while implementing the backend for UAPP Push Provisioning.

Your backend needs to:

1. Receive the `serverSessionId`, `opaquePaymentCard`, and `walletId`
2. Call Google's UAPP API with these parameters
3. Use Google's encryption keys to generate the Google OPC
4. Return both the TSP OPC and Google OPC to your app

Example backend response:
```json
{
  "tspOpc": "original-opaque-payment-card-from-tsp",
  "googleOpc": "encrypted-google-opaque-payment-card"
}
```

## Related Data Types

### `onPaymentCredentialsRequestPayload`

Payload received when Google Wallet requests payment credentials during UAPP provisioning:

```typescript
{
  requestId: string;           // Unique identifier for this request
  serverSessionId: string;     // Session ID from Google (crucial for backend)
  walletId: string;            // Google Wallet ID
  opaquePaymentCard: string;   // Original OPC from TSP
}
```

### `AndroidPaymentCredentialsResponse`

Response object containing encrypted payment credentials for UAPP flow:

```typescript
{
  opaquePaymentCard: string;         // TSP Opaque Payment Card
  googleOpaquePaymentCard: string;   // Encrypted Google OPC
}
```

## Required Data Elements

For UAPP provisioning, you need:

- **Opaque Payment Card** (OPC) - a binary blob of information Google Pay receives from the issuer app that could be presented to TSP to receive a token.
- **Google Opaque Payment Card** (Google OPC) - an encrypted version of the OPC required for UAPP (User Account Payment Provisioning) flow. This is generated by your backend using Google's encryption keys during the provisioning process.

## API Reference

### `addCardToGoogleWallet()`

Initiates the Google Wallet card provisioning flow with UAPP support.

**Parameters:**
- `cardData: AndroidCardData` - Card information required for provisioning
- `handlePaymentCredentialsGeneration: (data: onPaymentCredentialsRequestPayload) => Promise<AndroidPaymentCredentialsResponse>` - Handler function that receives the payment credentials request and returns encrypted credentials from your backend

**Returns:** `Promise<TokenizationStatus>` - Status of the tokenization process ('success', 'canceled', or 'error')

**Platform:** Android only

**Example:**
```typescript
const status = await addCardToGoogleWallet(
  cardData,
  async (data) => {
    const credentials = await fetchCredentialsFromBackend(data);
    return credentials;
  }
);
```

### AndroidCardData Optional Fields

When using UAPP with `addCardToGoogleWallet()`, the `AndroidCardData` object supports two optional boolean fields:

- **`isVirtualCard?: boolean`** - Set to `true` if provisioning a virtual card (digitally-generated card number linked to a physical card account). Virtual cards provide enhanced security for online transactions.

- **`isBounceProvisioned?: boolean`** - Set to `true` if the card was initially provisioned through Google Wallet and is now being completed through your app. This is used in bounce provisioning flows where users start in Google Wallet and are redirected to your app for additional verification.

For more details, see the [AndroidCardData Optional Fields](../README.md#androidcarddata-optional-fields) section in the main documentation.

## Testing

> [!WARNING]
> **UAPP implementation must be tested before production use.** Failure to provide required test evidence may result in revoking access to your production environment. Please make sure your implementation is thoroughly tested and approved by Google.

When implementing UAPP, ensure your app meets all Google testing criteria:

1. **UAPP Flow Testing** - Verify that payment credentials are correctly generated and transmitted during provisioning
2. **Backend Integration** - Ensure your backend correctly generates Google OPC using Google's encryption keys
3. **Error Handling** - Test timeout scenarios and credential validation failures
4. **Security Compliance** - Verify secure handling of encrypted payment credentials

For comprehensive testing requirements, refer to:
- [Google Push Provisioning Testing Criteria](https://developers.google.com/pay/issuers/apis/push-provisioning/android/test-cases)

## Troubleshooting

### Provisioning fails with timeout

If the backend request takes too long, Google Wallet may timeout. Ensure your backend responds within a reasonable timeframe (typically < 10 seconds).

### Invalid credentials error

Verify that:
1. Your backend is correctly generating the Google OPC using Google's encryption keys
2. Both `opaquePaymentCard` and `googleOpaquePaymentCard` are returned in the response
3. The credentials match the card being provisioned
